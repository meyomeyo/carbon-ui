CarbonUI Documentation

CarbonUIはシンプルで軽量なJavaScript UIライブラリです。jQueryライクなメソッドチェーンを用いながら、モダンな状態管理やSPAルーティング、各種ジェスチャーをサポートします。

API リファレンス目次

  • 1. Core (取得・ライフサイクル)
  • 2. CarbonWidget (単一要素操作)
  • 3. CarbonEls (複数要素操作)
  • 4. State (状態管理)
  • 5. Store (データ保存)
  • 6. Router (SPA遷移)

1. Core API (要素取得・ライフサイクル)

carbon.* オブジェクトから呼び出せる基本メソッドです。

要素の取得

メソッド戻り値説明
carbon.id(id)CarbonWidget指定したID(#不要)の要素を取得します。
carbon.class(className)CarbonEls指定したクラス(.不要)を持つ全ての要素を取得します。
carbon.find(selector)CarbonWidgetCSSセレクタで最初に一致する要素を取得します。

carbon.ready(callback)

DOMの構築(DOMContentLoaded相当)が完了したタイミングでコールバックを実行します。既に完了している場合は即座に実行されます。

carbon.ready(() => console.log('Ready!'));

2. CarbonWidget (単一要素の操作)

carbon.id() や carbon.find() で取得した要素は CarbonWidget としてラップされ、以下のメソッドをチェーンで利用できます。

プロパティ

  • widget.raw - ラップされている元の生DOM要素 (HTMLElement)
  • widget.state - 状態管理(State)オブジェクトへのアクセス

DOM操作・表示

メソッド説明
.show()要素の display: none を解除し表示します。
.hide()要素に display: none を設定して隠します。
.toggle()表示/非表示を切り替えます。
.isVisible()要素が表示されているか (boolean) を返します。
.isHidden()要素が隠れているか (boolean) を返します。
.text(val)要素のテキスト(textContent)を書き換えます。
.html(val)要素内のHTML(innerHTML)を書き換えます。
.destroy()DOMから要素を完全に削除します。イベントも解除されます。

入力・フォーム補助

メソッド説明
.disable()ボタン等に disabled 属性を付与します。
.enable()disabled 属性を解除します。
.isEmpty()input/textareaにおいて、入力値が空かどうか(boolean)を返します。
.isFilled()入力値が存在するか(boolean)を返します。

イベントとジェスチャー

メソッド説明
.tapped(fn)クリックまたはタップ(touchstart)で発火します。引数: (widget, event)
.focused(fn)フォーカス(focus)された際に発火します。
.unfocused(fn)フォーカスが外れた(blur)際に発火します。
.longPressed(fn, ms)長押しで発火します。第2引数で長押し判定のミリ秒(デフォルト500)を指定可能。
.swiped(fn, threshold)スワイプした際、方向("up"|"down"|"left"|"right")を第2引数として返します。第2引数で判定距離(デフォルト50px)を指定可能。

監視 (Observer)

.watch(prop, callback)

要素の属性や値の変更をリアクティブに監視します。
prop === 'value' の場合はinputなどの入力変更を監視し、それ以外の場合は data-* などのDOM属性の変動を監視します。

carbon.id('username').watch('value', (newValue) => {
    console.log(newValue);
});
.unwatch(prop)

指定したpropの監視(MutationObserver/EventListener)を解除します。

3. CarbonEls (複数要素の操作)

carbon.class() で取得した要素リストを扱うラッパーです。取得した複数の要素に対して、自動的に一括処理を行います。

メソッド説明
.each(fn)各要素(CarbonWidget)ごとにループ処理を実行します。コールバック引数: (widget)
.first()リスト内の最初の要素を CarbonWidget として返します。
.count()取得できた要素の数を返します。
.tapped(fn)全要素に対してtappedイベントを一括でバインドします。
.show()全要素を一括で表示します。
.hide()全要素を一括で非表示にします。

4. State (状態管理API)

CarbonWidget の .state プロパティからアクセスする状態管理特化のAPIです。
状態を適用すると自動的に carbon-[name] クラスと data-carbon-state="[name]" 属性がDOMに付与され、スタイルの切り替えや状態把握がスムーズになります。

メソッド説明
.apply(name)指定した状態(CSSクラスとdata属性)を適用します。
.remove(name)指定した状態を解除します。
.is(name)現在その状態が適用されているか (boolean) を返します。
.current()現在適用されている状態名(data-carbon-state)を文字列で返します。
.clear()付与されているすべての carbon-* 状態を初期化（削除）します。
carbon.id('btn').state.apply('loading'); // `carbon-loading` クラスを要素に付与
console.log( carbon.id('btn').state.current() ); // "loading" を出力

5. Store (Web Storage 操作)

carbon.store オブジェクト経由で sessionStorage への読み書きを安全に行います。
オブジェクトや配列は自動で JSON エンコード/デコードされ、キーの重複を防ぐため自動的に carbon_ プレフィックスが付与されます。

メソッド説明
store.set(key, val)データを保存します。オブジェクトや配列もそのまま渡せます。
store.get(key)keyに対応するデータを取得(デコード)して返します。無い場合は undefined。
store.delete(key)保存されたデータを削除します。

6. Router (SPAルーティング)

Fetch APIとHistory APIを活用した超軽量ルーター機能です。ページをリロードせずにコンテンツを切り替え、キャッシュを活用してサクサク動作させます。

carbon.init(options)

SPAルーターを初期化します。対象となるマウント領域(root)と初期ページ(home)を指定します。

carbon.init({
    root: '#app',          // 描画先DOM (デフォルト: '#app')
    home: 'component/home' // アクセス時に最初に読み込むパス (.htmlは不要)
});
carbon.go(pageName, replace = false)

指定したページ([pageName].html) を非同期(fetch)で取得し、指定したroot要素内を書き換えます。
取得したHTMLはキャッシュされ、内部の <script> タグは自動的に再評価・実行されます。
replace = true にするとブラウザに履歴(History)を追加せず上書き遷移します。

carbon.back()

ブラウザの履歴(History API)を利用して前の状態(一つ前のページ)へ戻ります。
